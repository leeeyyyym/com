데이터 과학은 숫자를 많이 다루는 과목이 아니라, 현실의 문제를 데이터로 이해하고, 근거 있게 설명하며, 더 나은 결정을 내리는 방법을 배우는 과목입니다.

질문무엇이 궁금한가?
분석데이터가 말하는 것은?
판단근거 있는 결론은?
오늘의 핵심 문장

“느낌”으로 말하기보다, 데이터를 보고 왜 그렇게 생각하는지 설명할 수 있어야 합니다.

이 과목에서 배우는 4가지 흐름

데이터 과학은 데이터의 의미를 이해하는 것에서 시작해, 준비·분석·모델링·프로젝트로 확장됩니다.

🔎

1. 데이터 과학의 이해

데이터가 사회와 의사 결정에서 어떤 가치를 갖는지 살펴봅니다.

🧹

2. 데이터 준비와 분석

데이터를 수집하고 정리한 뒤, 표·그래프·통계로 의미를 찾습니다.

📈

3. 모델링과 평가

예측·분류 같은 모델을 만들고, 결과가 타당한지 평가합니다.

🧩

4. 데이터 과학 프로젝트

팀별로 현실 문제를 정하고 데이터 기반 해결 과정을 발표합니다.

첫 시간 활동

오늘은 어려운 코딩보다 “데이터 과학적으로 생각하는 방식”을 먼저 경험합니다.

활동 1. 나의 하루 데이터 찾기

스마트폰 사용 시간, 수면 시간, 등교 시간, 음악 재생 기록처럼 내가 매일 남기는 데이터를 찾아봅니다.

  • 어떤 데이터가 만들어지는가?
  • 그 데이터로 어떤 질문을 만들 수 있는가?
  • 개인정보 보호가 필요한 부분은 무엇인가?

활동 2. 데이터 질문 만들기

관심 있는 주제를 하나 고르고, 데이터로 확인 가능한 질문으로 바꿔 봅니다.

  • “학생들은 잠을 적게 잔다” → 어떻게 확인할까?
  • “급식 만족도는 날씨와 관련 있을까?”
  • “공부 시간과 성취도는 항상 비례할까?”
첫 시간 출구 ticket : 오늘 수업이 끝나기 전, “내가 데이터로 확인해 보고 싶은 질문 1개”와 “그 질문에 필요한 데이터 2개”를 적어 제출합니다.

한 학기 수업 흐름

수업은 설명을 듣고 끝나는 방식이 아니라, 직접 데이터를 다루며 결과를 해석하는 방식으로 진행됩니다.

STEP 1

문제 발견

일상·학교·사회 현상에서 데이터로 확인 가능한 질문을 만듭니다.

STEP 2

데이터 준비

데이터를 수집하고, 결측값·이상값·표현 형식 등 분석 전 처리 과정을 배웁니다.

STEP 3

분석과 시각화

표, 그래프, 기초 통계, 시각화 도구를 활용해 데이터의 패턴을 읽습니다.

STEP 4

모델링과 평가

단순 예측·분류 모델을 만들어 보고, 모델이 믿을 만한지 기준을 세워 평가합니다.

STEP 5

프로젝트 발표

팀별로 데이터 기반 결론을 정리하고, 한계와 윤리적 고려까지 함께 발표합니다.

평가에서 중요하게 보는 것

정답 하나를 맞히는 것보다, 데이터로 문제를 해결하는 과정과 설명 능력을 봅니다.

문제 설정 질문이 데이터로 확인 가능한가?
데이터 처리 자료를 적절히 수집·정리했는가?
분석과 해석 그래프와 통계를 근거로 설명했는가?
모델 평가 결과를 무조건 믿지 않고 한계를 따졌는가?
협업과 기록 역할을 나누고 과정 기록을 남겼는가?
윤리와 책임 개인정보·편향·저작권을 고려했는가?

※ 실제 반영 비율은 학교 평가계획에 맞게 수정하세요.

수업 약속

데이터 과학 수업에서는 결과보다 태도가 중요합니다.

원하는 결론에 맞추기 위해 데이터를 숨기거나 바꾸지 않습니다.
이름, 연락처, 민감 정보는 수집·공유하지 않습니다.
“그냥 그런 것 같다”보다 “데이터를 보면…”으로 설명합니다.
데이터가 부족하거나 편향될 수 있음을 함께 생각합니다.
